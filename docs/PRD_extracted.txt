1. Product Overview & Vision
What we're building, why it matters, and who we're building it for
Vision: Ek aisa AI-powered platform jo har Indian student aur professional ko recruiter-ready resume banane mein help
kare — bina career counselor ke.
1.1 Mission Statement
ResumeAI ka mission hai har job seeker ko — chahe fresher ho, career switcher ho, ya non-English speaker — ek
intelligent AI coach dena jo unka resume real-time mein improve kare, ATS systems se paar karaye, aur unhe interview
tak pohchaye.
1.2 Problem Statement
India mein har saal 1.5 crore+ graduates job market mein aate hain. In mein se 78% ke paas professionally written
resume nahi hota. Current resume builders: templates dete hain lekin guidance nahi. ATS rejection rate 75% se zyada
hai. Professional resume writers expensive hain (■2,000–■15,000). Non-English speakers ke liye aur bhi mushkil hai.

4. Core Features — Deep Dive
Every feature explained with implementation detail
4.1 AI Resume Builder
 AI-guided section-by-section creation flow (not a blank form)
 AI Questioning System: asks user about experience, projects, achievements via chat interface
 Extracts structured data from conversational answers using GPT function calling
 Auto-fills resume sections from user's chat responses
 Smart defaults based on role/industry/experience level selection
4.2 Real-Time AI Suggestions
 Inline suggestions appear as user types (500ms debounce)
 Weak phrase detection: flags 'responsible for', 'helped with', 'worked on'
 Strong verb suggestions: 'Architected', 'Reduced', 'Scaled', 'Delivered'
 Hinglish coaching: 'Yeh bullet weak hai — isko quantify karo!'
 Context-aware: different suggestions for fresher vs senior profiles
4.3 ATS Score Checker
 Analyzes resume against 40+ ATS criteria (keywords, formatting, structure)
 Scores: Overall ATS Score, Keyword Match %, Format Score, Readability
 Identifies: missing sections, problematic formatting, keyword gaps
 Section-level scoring — shows which section is pulling score down
 Actionable fix suggestions for each failing criterion
4.4 JD Keyword Matching & Tailoring
 User pastes or uploads Job Description
 AI extracts must-have and nice-to-have keywords from JD
 Compares against resume — shows matched vs missing keywords
 One-click tailoring: AI rewrites relevant bullets to include missing keywords naturally
 Tailoring score: shows % match improvement after each edit
4.5 AI Bullet Improvement
 Single-click bullet rewrite using STAR format (Situation → Task → Action → Result)
 Adds metrics and quantification where user inputs raw data
 Three rewrite variants shown — user picks best one
 Before/After comparison UI
 Explains WHY the new version is better (educational mode)

5. AI System Design & Architecture
Micro-AI architecture, prompt engineering, and cost optimization
5.1 Micro-AI Architecture
ResumeAI uses a modular micro-AI architecture where each feature has its own dedicated AI pipeline with specialized
prompts, context, and output parsers. This prevents prompt interference and enables independent cost optimization.

5.2 Prompt Engineering Strategy
 System Role Injection: Each AI module has a specialized system prompt that establishes expert persona (e.g., 'You
are a senior tech recruiter at a Fortune 500 company...')
 Structured JSON Output: All AI responses are requested as JSON with strict schema — prevents hallucination and
enables reliable parsing
 Few-Shot Examples: Each prompt includes 2-3 example input/output pairs for consistent formatting
 Context Passing: Resume sections passed as structured JSON context, not raw text
 Temperature Control: Creative tasks (roast, suggestions): temp 0.7 | Scoring/ATS: temp 0.1 for consistency
 Token Budget Enforcement: Max tokens set per module to control costs — 200 for bullets, 1000 for full analysis
5.3 Context Memory Strategy
ResumeAI maintains session-level context across the resume building conversation. A rolling context window stores the
last 10 exchanges + current resume state. Redis cache stores session context (TTL: 2 hours). Full context saved to
MySQL ai_sessions table for premium users' history feature.
5.4 Hallucination Prevention
 AI never invents job titles, companies, or dates — only rewrites user-provided content
 Strict JSON schema validation on all AI outputs — malformed responses trigger retry
 Achievement metrics only suggested when user provides a base number to quantify
 All AI suggestions shown as 'suggestions' — user must explicitly accept changes

6. UX Flow & User Journey
Complete interaction flows for every major user path
6.1 Onboarding Flow
Step 1: Landing page → 'Build Free Resume' CTA (no signup required for preview)
Step 2: Quick profile: Name, Target Role, Experience Level (3 options: Student / 0-2 yrs / 2+ yrs)
Step 3: Template selection (3 recommended based on profile)
Step 4: Email signup shown AFTER first section is filled (value-first onboarding)
Step 5: Google OAuth or Email/Password — JWT token issued
Step 6: User lands on Resume Editor with pre-filled name and role
6.2 Resume Creation Flow
Step 1: Left: Section navigator (Contact, Summary, Experience, Education, Skills, Projects)
Step 2: Center: Live resume preview (real-time rendering)
Step 3: Right: AI Coach panel (suggestions, tips, score)
Step 4: Each section has AI-guided mode (questionnaire) or manual mode
Step 5: AI questionnaire: 'Tell me about your most impactful project' → extracts structured data
Step 6: Progress bar shows resume completeness score
Step 7: Auto-save every 30 seconds to backend
6.3 AI Coaching Interaction Flow
Step 1: User types in experience bullet → 500ms debounce → AI analyzes
Step 2: AI suggestion appears as inline ghost text OR right panel card
Step 3: User can: Accept (Tab key), Dismiss (Esc), See 3 alternatives, Ask why
Step 4: 'Why is this better?' shows educational explanation in Hinglish
Step 5: Weak phrases highlighted in yellow underline — click to get improvement
Step 6: AI Coach chat at bottom: user can ask 'How do I describe my internship at TCS?'
6.4 JD Upload & Tailoring Flow
Step 1: User clicks 'Tailor for Job' → paste JD or upload PDF
Step 2: AI extracts: Required Skills, Preferred Skills, Role Keywords, Company Values
Step 3: Side-by-side view: Resume keywords vs JD keywords
Step 4: Missing keywords shown in red — click each to see suggested placement
Step 5: One-click 'Auto-Tailor' rewrites bullets to include top 5 missing keywords
Step 6: Before/After comparison with match score improvement shown

11. Security & Privacy
11.1 Authentication & Authorization
 JWT Access Tokens: 15-minute expiry — short-lived to limit exposure
 Refresh Tokens: 7-day expiry, stored httpOnly cookie — not accessible to JS
 Password Hashing: bcrypt with cost factor 12
 OAuth Security: PKCE flow for Google OAuth — prevents auth code interception
 Rate Limiting: express-rate-limit: 5 login attempts / 15 min per IP — blocks brute force
 Role Guards: Middleware validates subscription tier before AI feature access
11.2 Data Security
 All API communication over HTTPS (TLS 1.3)
 Resume content stored as JSON in MySQL — no raw file storage
 Uploaded JD files scanned for malware before processing — multer + file-type validation
 User photos / avatars stored with randomized filenames — no enumerable URLs
 SQL injection prevention: Sequelize ORM with parameterized queries throughout
 XSS prevention: helmet.js headers, DOMPurify on all user-rendered content
11.3 AI Privacy
 Resume data sent to OpenAI API — only the specific section being analyzed, never full PII
 No user data used for OpenAI model training (Enterprise API agreement)
 User can request 'AI amnesia' — clears all stored AI feedback from DB
 AI suggestions are ephemeral by default — only stored if user accepts
